# Git 運用

## ブランチ

- 作業は原則 `main` ブランチで行い、作業用ブランチは作成しない。
- デフォルトブランチも `main` とし、ブランチを散らかさない。
- `main` への直接 push が制限されている場合でも、このプロジェクトでは直接 push を許可する。
- 実行環境の制約で直接 push できない場合は、制約を明示し、勝手に運用方針を変えない。
- やむを得ず作業用ブランチを作成した場合は、役目を終えたら削除する。

## コミット

- コミットメッセージは日本語で、淡々と変更内容のみを書く。
- 変更前に差分を確認する。
- 秘密情報をコミットしない。

## 禁止事項

- コミット・Issue・PR に `by Claude` のような文言や AI の宣伝文句を含めない。
- コミット履歴や Issue / PR 内に、依頼者へのアドバイスや質問を含めない。
