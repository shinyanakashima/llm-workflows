# プロダクト面の好み

## 対外性・可読性

- 日英バイリンガル対応を重視する。
- 対外的な読みやすさ、国際性、第三者可読性を重視する。
- ドキュメントは第三者が読んで分かる内容にする。

## README の構成

- README の冒頭に、公開 URL と「何のためのものか」「誰向けか」を明示する。
- 必要に応じてスクリーンショットを README の前段に置く。
- 開発者向け情報、再現手順、構成、前処理方針、デプロイ方法は後段にまとめる。

## 地図アプリの説明

- 地図アプリの場合は、地図の見方を平易に説明する。
- 円・色・大きさ・クリック操作・日英切替などの UI 説明を入れる。
- 指標の意味を表で整理する。
- 数字を読むときの注意を独立セクション化する。
- 用語説明を追加する。
