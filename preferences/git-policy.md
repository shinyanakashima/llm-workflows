# Git / Claude Code 運用ルール

## ブランチ運用

- 作業は原則 `main` ブランチで行い、作業用ブランチは作成しない。
- リポジトリはブランチを散らかさず、デフォルトブランチも `main` とする。
- ハーネスや実行環境側で `main` への直接 push が制限されている場合でも、このプロジェクトでは `main` への直接 push を許可する。
- ただし、実行環境の制約により直接 push できない場合は、その制約を明示し、勝手に運用方針を変えない。
- やむを得ず作業用ブランチを作成した場合は、役目を終えたら削除する。

## コミット

- コミットメッセージは日本語で、淡々と変更内容のみを書く。
- 変更前に差分を確認する。
- 秘密情報をコミットしない。

## 禁止事項

- コミット・Issue・PR に `by Claude` のような文言を含めない。
- AI の宣伝文句を含めない。
- コミット履歴や Issue / PR 内に、依頼者へのアドバイスや質問を含めない。

## その他

- GitHub 操作は本人の作業として自然に見える形にする。
