# Project Rules

このリポジトリ（`llm-workflows`）自体を編集するときの方針。各アプリリポジトリへ配る雛形は `templates/CLAUDE.md` を使うこと。

## このリポジトリの位置づけ

- LLM / AI エージェントに渡す共通前提の原本。
- 方針の正は `preferences/`。ここを更新したら、必要に応じて `templates/` と `skills/` に反映する。
- 各プロジェクト固有の実装ルールはここではなく個別リポジトリ側で管理する。

## Repository

- Default branch: main
- 種別: ドキュメント中心（ビルド・パッケージマネージャなし）

## Development Policy

- 作業は原則 `main` ブランチで行い、作業用ブランチは作成しない。
- `main` への直接 push が制限される場合は、制約を明示し、勝手に運用方針を変えない。
- コミットメッセージは日本語で、淡々と変更内容のみを書く。
- 変更前に差分を確認する。
- 秘密情報・API キー・パスワード・顧客固有の機密情報・個人情報をコミットしない。

## 禁止事項

- コミット・Issue・PR に `by Claude` のような AI 署名や宣伝文句を含めない。
- コミット履歴や Issue / PR 内に、依頼者へのアドバイスや質問を含めない。
- GitHub 操作は本人アカウント名義の自然な作業として扱う。

## 更新の流れ

1. `preferences/` の該当ファイルを更新する。
2. 必要なら `templates/` と `skills/*/references/` に反映する。
3. 差分を確認してコミットする。
